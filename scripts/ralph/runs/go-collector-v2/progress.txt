## Codebase Patterns
- Go binary 位于 packages/logger-collector/，零外部依赖，只用 Go stdlib
- server 包在 internal/server/，负责 HTTP 和启动输出
- query 包在 internal/query/，负责 read/filter/format
- main.go 负责 CLI 路由和参数解析
- Node 薄壳在 packages/logger/src/cli.ts，零外部依赖
- quality-gate 的品牌规范：templates/banner.txt（ASCII art）+ 5 色渐变（38;5;75/105/141/175/204）
- quality-gate 的 ECAM 面板：C 颜色常量 + ╔═╗ box-drawing + dim green 边框 + 内宽 60

  - Help text uses **programmatic ANSI colors** (C.Wht/C.Grn/C.Gry) even though quality-gate's help.txt is plain text, per PRD AC requirement
  - New file `internal/banner/help.go` with `PrintHelp(w io.Writer, prefix string)` function
  - Include BOT border fix from US-001 tester observation (╘╙ -> ╚╝, U+2558/2559 -> U+255A/255B)
- Wrote comprehensive product handoff at `scripts/ralph/runs/go-collector-v2/handoff/product.md` covering:
  - Exact help text template with all sections and labels
  - invokePrefix substitution rules (dynamic in Usage, hardcoded in Examples)
  - ANSI color scheme: titles=bright white, labels=green, descriptions=light gray
  - How to replace flag.Usage with early interception in detectSubcommand
  - Structured rendering approach (helpSection/helpLine types)
  - BOT border character fix specification
  - Acceptance criteria mapping (6 ACs)
  - Files to change (3 files: help.go new, panel.go fix, main.go modify)
- Updated prd.json: US-002 `handoff.product = true`, `stage = "developer"`
- **Learnings for future iterations:**
  - quality-gate's `renderHelp()` does NOT color the help text itself -- only the banner is colored. The PRD adds coloring as an enhancement.
